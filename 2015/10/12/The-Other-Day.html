The Other Day I Had an Idea

Oct 12, 2015

Saturday, October 11th. As usual, I lay mulling over ideas instead of going to sleep. Today’s topic was journaling and accountability. I’ve been meaning to start a journal for some time, but the time has never felt right. So I had a wonderful, terrible, idea: I would use GitHub to start a blog! Here is the story of how I worked on this all Sunday afternoon and evening without writing a single post.

I’ve only really started using GitHub in the last three days. I found the Open Source Society’s Computer Science path on Friday via a Reddit post, and pushed got me to finally make an account. A few hours later, and voilà! My first repository! Creating the repo actually took maybe 5 minutes tops, including tutorials. The bulk of my time was spent writing scripts to make folders, writing different scripts to rename folders after I realized I misspelled all the names, and figuring out why my commits weren’t being credited to me. Hint: useres.github.io.

As you might have surmised by this point, I am interested in programming and computer science. Other pursuits of mine include physics, music, mathematics, books, and I am currently in the process of learning how to draw a straight line.

Back to the story. On Sunday morning I woke up and decided that I want to use Jekyllfor my blog. Its website tells me I need Ruby, RubyGems, a Unix-based OS, NodeJs, and Python. To my pleasant surprise, I already had Ruby installed.

That wasn’t the last surprise on Sunday. Just the last pleasant one.

Step 1: Install with RubyGems

$ gem install jekyll'

Nothing happens.

C^c

Eventually, I find out that gem runs in silent mode by default. I disable that, and see lots of html status codes. Apparently, the server that gem downloads files from moved all their files and left redirects in their place. So that adds up to 3 lines of output for ever file GEM tries to download. After waiting for around a minute, gem quits with an error. It helpfully tells me to run brew doctor. Crap. A couple months ago I screwed up my installation of homebrew beyond belief.

My memory gets fuzzy at this point. It’s fractured and bits might be suppressed. I vaguely remember having to edit my sudoers file. I definitely had to install the Ruby Version Manager at some point, because once I took care of the other problems I found that I was using Ruby 1.8.3. Despite my Git installation being squeaky clean until Friday, I found permission errors within it.

By midnight I was hacking together a script that builds the blog, copies it to a repo, and pushes the repo to GitHub. I ended the evening with a function that opens a new blog post in Vim, using a predefined template.

Which brings me to the end of today’s post.

Hopefully my formatting worked.
